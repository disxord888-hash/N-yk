# hyk サイズ命名規則 仕様書

## 1. 概要

`hyk` は、既存の `yk` サイズ命名規則を拡張した **半指数（half yk）サイズ体系**である。

* 表記形式：`Nhyk`
* 意味：`2^N × 1.5 bytes`
* `h` は **half（中間段階）** を意味する
* `yk` と `yk+1` の **ちょうど中間サイズ**を表現することを目的とする

例：
```text
8hyk = 2^8 × 1.5 bytes = 256 × 1.5 = 384 bytes
```

---

## 2. 命名思想

* 2の累乗体系の分かりやすさを保ったまま、粒度を1段階細かくする
* 連続性を崩さず、指数的スケールを維持する
* 人間が直感的に「yk と次の yk の中間」と理解できる

---

## 3. 基本定義

```text
Nhyk = 2^N × 1.5 bytes
```

* `N` は整数
* 基数は常に 2
* 係数は固定で **1.5**
* 小数指数や可変係数は認めない

---

## 4. yk / hyk の関係

| 種類      | 定義              |
| ------- | --------------- |
| Nyk     | 2^N bytes       |
| Nhyk    | 2^N × 1.5 bytes |
| (N+1)yk | 2^(N+1) bytes   |

関係式：
```text
Nyk < Nhyk < (N+1)yk
```

## 11. 拡張子としての使用ルール

* `.Nhyk` は **最大許容サイズが `2^N × 1.5` バイトであることを示す**
* `.Nyk` と組み合わせることで、
  * yk → hyk → (N+1)yk
    という **3段階サイズ階層**を構成できる
